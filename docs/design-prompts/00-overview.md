# デザイン差別化 概要

## 目的

3つのツールが「色違いの同じデザイン」になっている問題を解決し、各ツールに独自のデザイン言語を確立する。

---

## 3ツールの差別化サマリー

| 項目 | 携帯プラン最適化 | クレカ明細チェッカー | 不動産仲介手数料 |
|------|-----------------|-------------------|-----------------|
| **テーマ** | テック・比較・スマート | 家計簿・温かみ・気づき | プロフェッショナル・信頼 |
| **主役** | 比較テーブル | グラフ・ビジュアル | 診断レポート・CTA |
| **トーン** | 効率的・データドリブン | 親しみやすい・発見 | フォーマル・安心 |
| **アクセント色** | シアン (#0891B2) | コーラル (#E07A5F) | ネイビー (#1E3A5F) |
| **背景色** | 純白系 (#FAFAFA) | クリーム系 (#FBF8F3) | クールグレー (#F5F6F8) |
| **角丸** | 8px（シャープ） | 16-20px（ソフト） | 8px（フォーマル） |
| **シャドウ** | なし/極薄ボーダー | 温かみのある影 | 薄くシャープ |
| **特徴的UI** | 比較スライダー、テーブル | カテゴリカード、円グラフ | ゲージメーター、LINE CTA |

## 実装の進め方

### Phase 1: カラーパレット変更
各ツールの`tailwind.config.js`を更新

### Phase 2: 結果画面のリデザイン
最もユーザーの目に触れる結果画面から着手

### Phase 3: 入力画面の調整
フォームやアップロード画面のスタイル調整

### Phase 4: マイクロインタラクション
アニメーションや細かいUX改善

---

## 注意事項

1. **共通パッケージは変更しない**
   - `packages/shared/`のコンポーネントはそのまま
   - 各ツールで上書き or 独自コンポーネント作成

2. **機能は変えない**
   - あくまでデザイン変更のみ
   - ロジックやデータ構造は触らない

3. **モバイルファースト維持**
   - スマホでの見た目を最優先
   - タップ領域44px以上は維持
