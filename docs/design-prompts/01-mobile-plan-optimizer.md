# 携帯プラン最適化ツール デザイン差別化プロンプト

## 背景

現在、6億ニキツール群には3つのツールがある：
1. 携帯プラン最適化（このツール）
2. クレカ明細チェッカー
3. 不動産仲介手数料チェッカー

**問題**: 3つのツールが「色違いの同じデザイン」になっており、ユーザーから見ると差別化ができていない。カード、ボタン、レイアウト構造がすべて同一で、アクセントカラーだけが異なる状態。

**目標**: 各ツールのテーマ・世界観に合わせた独自のデザイン言語を確立し、一目で「別のツール」とわかるようにする。

---

## このツールのコンセプト

### テーマ: 「テック・比較・スマート」

携帯プランは**データと比較**が主役。ユーザーは複数のプランを見比べて、自分に最適なものを選びたい。

**キーワード:**
- 比較表・一覧性
- データドリブン
- シンプル・効率的
- テック感（でもAIっぽくなりすぎない）

**ペルソナ:**
- 20〜40代
- 「今のプランが高い気がする」
- サクッと比較して決めたい
- 細かい説明より結果重視

## デザイン方針

### 1. レイアウト: 比較テーブルを主役に

```
【現状】カード型で1プランずつ縦に並べる
【変更】比較テーブル形式で横並び表示を増やす

- 結果画面では2〜3プランを横並びで比較できるUIに
- 「おすすめ」「現在」「次点」を並べて違いを可視化
- スマホでは横スクロールのテーブル or スワイプカルーセル
```

### 2. 色使い: クリーンなテック感

```
【現状】赤アクセント + オフホワイト（温かみ系）
【変更】よりクリーンでテック寄りに

- 背景: 純粋な白 or 極薄グレー (#FAFAFA)
- アクセント: 鮮やかな青緑系 (#0891B2 シアン系) に変更
- セカンダリ: 濃いグレー (#374151)
- 「節約額」など金額強調: 緑 (#10B981)
- 「高い」警告: オレンジ (#F59E0B)
```

### 3. カードスタイル: シャープ・ミニマル

```
【現状】角丸12px、ソフトシャドウ
【変更】よりシャープに

- 角丸: 8px（少し控えめに）
- シャドウ: なし or 極薄いボーダーのみ
- ボーダー: 1px solid #E5E7EB（明確な区切り）
- カード内余白: やや狭め（12px〜16px）
```

### 4. データ表示: 数字を目立たせる

```
- 金額は大きく太字（24px〜32px、font-bold）
- 単位は小さく（14px）
- 差額は色付きバッジで表示（-2,000円/月 → 緑背景）
- プログレスバーやメーターでデータ量を可視化
```

### 5. アイコン: テック系

```
- Lucide Reactを継続使用
- よく使うアイコン:
  - Smartphone, Signal, Wifi, Database
  - TrendingDown（節約）, TrendingUp（高い）
  - CheckCircle, XCircle（比較用）
  - BarChart3, PieChart（データ可視化）
```

### 6. 特徴的なUI要素

```
1. 比較スライダー
   - 「データ量」「通話」などをスライダーで調整
   - リアルタイムでおすすめプランが変わる

2. プラン比較カード
   - 左右スワイプで複数プラン比較
   - 「現在」「おすすめ」「次点」のラベル付き

3. 節約額ハイライト
   - 結果画面の最上部に大きく表示
   - 「年間 ¥24,000 節約」のような表示
   - 緑のアクセントで目立たせる

4. スペック比較テーブル
   - データ量、通話、料金を一覧表示
   - 行ごとに「勝ち」マークを表示
```

## 実装時の注意

1. **共通コンポーネント（packages/shared）は変更しない**
   - このツール専用のコンポーネントをsrc/components/に作成

2. **tailwind.config.jsでカラーを上書き**
   ```js
   colors: {
     'bg-base': '#FAFAFA',
     'bg-card': '#FFFFFF',
     'text-main': '#111827',
     'text-sub': '#6B7280',
     'accent': '#0891B2',      // シアン
     'accent-light': '#ECFEFF',
     'border': '#E5E7EB',
     'success': '#10B981',
     'warning': '#F59E0B',
   }
   ```

3. **既存機能は壊さない**
   - デザイン変更のみ、ロジックは触らない
   - 段階的に変更（まず結果画面から）
